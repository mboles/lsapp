Quick Start: Using FTP Batch Provisioning to delete assets

This Quick Start will show you how to delete a video asset and a playlist using FTP batch provisioning.

After completing this Quick Start you should be able to:

  • Delete video assets
  • Delete a playlist

Note: FTP Batch Provisioning is available only in Video Cloud Pro and Enterprise editions.

Hands-On Task: Deleting video and playlist assets

In this task, you will delete a video and playlist asset using FTP batch provisioning.

Create the XML manifest file

First, you will create the XML manifest file.

  1. Create an XML manifest file with the XML declaration. Include the <publisher-upload-manifest> root tag with a nested <notify> tag.

    2. Note: For details on how to create the XML manifest file with the root tag, refer to Quick Start: Using FTP Batch Provisioning to Upload Videos.

  2. Check to be sure your manifest file appears as follows, but with your respective values. 
    <?xml version="1.0" encoding="utf-8" ?>
    <publisher-upload-manifest publisher-id="1234567800001" preparer="MB FTP" report-success="true">
      <notify email="bclftp@gmail.com" />
      ...
    </publisher-upload-manifest>

Delete an asset

Next, you will delete an asset using the <delete-asset> tag.

  1. Add a <delete-asset> tag nested in the <publisher-upload-manifest> code block, and just below the <notify> tag.
  2. In this tag, you will use one attribute. Read the following as an introduction to their use.
    3. Attribute Required Description
    refid Required The refid of your asset to be deleted.
  3. Enter the following attribute/value pairs in the tag:
    • refid: woodpecker_asset_refid
  4. Your code should appear as follows: 
    <delete-asset refid="woodpecker_asset_refid"/>

    5. You can not delete an asset without deleting its associated title. You also can not delete an asset if it is referenced in a manual playlist. You can, however, delete an asset which is referenced in a smart playlist.

Delete a video

Next, you will delete a video using the <delete-title> tag.

  1. Add a <delete-title> tag nested in the <publisher-upload-manifest> code block, and just below the <delete-asset> tag.
  2. In this tag, you will use one attribute. Read the following as an introduction to their use.
    3. Attribute Required Description
    refid Required The title refid of your video to be deleted.
  3. Enter the following attribute/value pairs in the tag:
    • refid: woodpecker_title_refid
  4. Your code should appear as follows: 
    <delete-title refid="woodpecker_title_refid"/>

Delete a playlist

Next, you will delete a playlist using the<delete-lineup> tag.

  1. Add a <delete-lineup> tag nested in the <publisher-upload-manifest> code block, and just below the <delete-title> tag.
  2. In this tag, you will use one attribute. Read the following as an introduction to their use.
    3. Attribute Required Description
    refid Required The refid of your playlist to be deleted.
  3. Enter the following attribute/value pairs in the tag:
    • refid: woodpecker_title_refid
  4. Your code should appear as follows: 
    <delete-lineup refid="manualPlaylist1"/>
  5. Check to be sure your manifest file appears as follows, but with your respective values. 
    <?xml version="1.0" encoding="utf-8" ?>
    <publisher-upload-manifest publisher-id="1234567800001" preparer="MB FTP" report-success="true"
     <notify email="bclftp@gmail.com"/>
      <delete-asset refid="woodpecker_asset_refid"/>
      <delete-title refid="woodpecker_title_refid"/>
      <delete-lineup refid="manualPlaylist1"/>
    </publisher-upload-manifest>

Submit the XML manifest file to the Video Cloud server

Now, you will submit the XML manifest file to delete your video assets and your manual playlist.

  1. Upload the XML manifest file to the Brightcove server using an FTP client.
  2. Wait for the processing to complete. The XML manifest file should disappear from your FTP client. You may have to refresh the view.

    3. By default, you will receive error messages in an email. You will also receive success messages if you set the report-success attribute in the <publisher-upload-manifest> tag to a value of true.

  3. You can now go into the Video Cloud Studio and navigate to the Media module to see your updated video assets and your new manual playlist.
